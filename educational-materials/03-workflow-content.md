# Module 3: Making Changes Workflow Content Specification

## Overview
This 8-slide module teaches the core Edit→Save→Commit→Deploy workflow that forms the foundation of all website updates.

## Slide 1: Your First Website Update Adventure!
**Title:** "Your First Website Update Adventure! 🌟"
**Content:** Introduction to making website changes, what they'll accomplish (text change), encouragement that this is the core skill for all future updates. Bible verse: Proverbs 16:9 about planning and God establishing steps. Safety reminder about backups.

## Slide 2: The Magic Website Workflow
**Title:** "The Magic Website Workflow ✨"
**Visual:** Circular workflow diagram showing Edit→Save→Commit→Deploy cycle
**Content:**
- Edit: Use Claude Code to make changes with AI help
- Save: Changes saved locally on computer
- Commit & Push: Send to GitHub backup with description
- Live: Vercel automatically publishes to live site (2-3 minutes)
**Key Message:** This pattern works for ALL website updates

## Slide 3: Opening Your Website Workshop
**Title:** "Opening Your Website Workshop 🛠️"
**Content:** Step-by-step instructions for opening the website folder in Claude Code:
1. Find nomatterhowsmall-website folder
2. Open with Claude Code
3. Wait for loading
4. Test Claude response with "Hi Claude! I'm ready to update my website"
**Troubleshooting:** Basic fixes for common opening issues

## Slide 4: Making Your First Text Change
**Title:** "Making Your First Text Change ✏️"
**Content:**
- Choose simple first change (headline, bio, quote)
- How to ask Claude: "Claude, I want to change the main headline to [new text]"
- What Claude will do: find location, show change, ask for approval
- Example conversation showing the process
**Key Point:** Claude handles the technical details, you focus on what you want

## Slide 5: Saving Your Changes (Commit to GitHub)
**Title:** "Saving Your Changes (Commit to GitHub) 💾"
**Content:**
- What 'commit' means (like diary entry with timestamp)
- How to ask Claude: "Claude, I'm ready to commit this change to GitHub"
- Claude suggests commit message, handles technical process
- Example commit messages: "Update main headline" or "Add new Bible verse"
**Safety:** Every commit creates a backup point

## Slide 6: Watching the Magic Happen (Live Site Update)
**Title:** "Watching the Magic Happen (Live Site Update) ✨"
**Content:**
- Timeline: GitHub (instant) → Vercel notified (30 sec) → Site built (1-2 min) → Live update (2-3 min total)
- How to check: Open nomatterhowsmall.life in browser, wait, refresh
- Success celebration: Take screenshot, share excitement
**Troubleshooting:** What to do if changes don't appear (wait, refresh, ask Claude)

## Slide 7: Troubleshooting Common Hiccups
**Title:** "Troubleshooting Common Hiccups 🔧"
**Content:** Problem-solving flowchart:
- Changes not showing: Wait 5 min → Refresh → Clear cache → Check different device → Ask Claude
- Want to undo: Tell Claude "I want to undo my last change"
- Claude not responding: Check internet → Restart app → Reopen folder
- Don't know what to change: Ask Claude for suggestions
**When to ask humans:** If troubleshooting steps don't work

## Slide 8: Workflow Mastery Celebration!
**Title:** "Workflow Mastery Celebration! 🎉"
**Content:**
- What they accomplished: Made first website change, learned core workflow, can now update any text
- New superpower: Can update ANY text on website anytime
- This workflow works for: Text changes, new content, Bible verses, bio updates
- Bible verse: Colossians 3:23 about working with all your heart
- Next adventures: Images (Module 4), Customizations (Module 5), Help (Module 6)
**Achievement:** 🏆 Website Update Master
